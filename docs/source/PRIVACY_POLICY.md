# Privacy Policy

> **Draft Notice:**  
> This document outlines the proposed privacy policy for MarketMind. It is currently a draft and not yet implemented. The practices described here are under review and may change. Until finalized, do not assume full adherence to the details in this policy.

**Version:** 1.0.0  
**Last Updated:** March 12, 2026

This Privacy Policy describes how the MarketMind application ("MarketMind", "we", "us", or "our") handles data. In short: MarketMind does not collect or process personal data about users. The application uses financial data from public sources or authorized market-data APIs to function, and all handling is intended to align with relevant privacy regulations.

## Data Collection

### No Personal Data Collected

MarketMind does not collect names, email addresses, postal addresses, or other personal identifiers. The application does not require user accounts or profile creation.

### Financial Data Only

The application accesses only market and financial data needed for research, analysis, and prediction workflows. This includes public market data, company and macroeconomic data, and data retrieved via third-party APIs such as Interactive Brokers.

### User-Provided API Credentials

If you connect MarketMind to an external data source, credentials are used only to establish that connection. They are not transmitted to the project maintainers or stored remotely by MarketMind.

### No Tracking or Analytics

MarketMind does not use tracking, analytics, cookies, browser fingerprinting, or telemetry services by default.

## Data Usage

### Purpose of Data Use

Retrieved financial data is used only to perform MarketMind’s core functions, such as model training, prediction, backtesting, and results display.

### Local Processing

All data processing happens locally on the user’s machine by default. MarketMind does not send user datasets or results to a developer-controlled cloud service for computation.

### No Secondary Use

Data is not used for marketing, advertising, or user profiling.

## Data Storage

### Local Storage

Downloaded market data, cached information, model files, logs, and outputs are stored locally by default. The maintainers do not have access to that local data.

### Data Encryption at Rest

By default, data stored by the application is not encrypted by MarketMind. Users should protect devices running MarketMind with appropriate local security controls such as strong authentication and full-disk encryption.

Future versions may offer optional application-level encryption for selected local artifacts, but that does not change the current default.

### No Remote Databases

MarketMind does not send user data to a centralized developer-operated database.

## Data Sharing

### No Third-Party Sharing

MarketMind does not sell, rent, or trade user data.

### No External Uploads

The application does not upload local research data to a developer-controlled server. Network activity is limited to user-requested access to external market-data providers.

### External APIs

When the application queries external providers such as Interactive Brokers or Yahoo Finance, only the minimum necessary request data is sent to those providers. Users should review the privacy practices of those services separately.

## Security

### Secure Connections

Where MarketMind communicates with external providers, it uses secure transport mechanisms such as HTTPS or provider-supported secure SDK paths.

### Data Security on Your Device

Because MarketMind stores data locally and does not encrypt it by default, device security remains important. Users should secure their local environment appropriately.

### Privacy by Design

MarketMind’s default local-first architecture is intended to minimize privacy risk by reducing personal-data collection and external data transfer.

## User Rights

Because MarketMind does not collect or centrally store personal data, most access, correction, deletion, and portability rights are handled by the user directly through their own local files and system controls.

- Right to be informed: this policy describes the project’s data-handling model
- Right of access and portability: local files remain under the user’s control
- Right to erasure: uninstalling the app and deleting local files removes locally stored data
- Right to object or restrict processing: users control whether they connect external data sources
- No sale of personal data: MarketMind does not monetize user data

## Changes to This Policy

This Privacy Policy may change as the project evolves or laws change. Significant changes should be reflected in updated documentation and release notes.
